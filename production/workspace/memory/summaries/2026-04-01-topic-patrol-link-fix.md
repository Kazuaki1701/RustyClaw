### 2. Warpターミナル：活用ガイド
Warpを最大限に活用するための「よく使う操作・手順・環境変数」のまとめです。

#### **① よく使う機能・コマンド**
*   **AIコマンド検索 (`Ctrl + `)**: 自然言語でやりたいことを入力（例：「ポート8080を使っているプロセスを殺す」）すると、コマンドを提案してくれます。
*   **ワークフロー (`Shift + Ctrl + W`)**: よく使う複雑なコマンドを保存し、引数付きで呼び出せます。
*   **ブロック操作**: 各コマンドの結果が独立した「ブロック」になります。`Command + C` で出力だけをコピーしたり、AIにエラーを解説させたりできます。
*   **セッション復元 (Launch Configuration)**: プロジェクトごとに複数のタブやウィンドウの配置を保存し、一発で復元できます。

#### **② 推奨される手順（ワークフロー）**
1.  **コマンド履歴の活用**: `Up Arrow` で履歴を検索し、ブロックを選択して `Command + Enter` で再実行。
2.  **マルチカーソル編集**: IDEのように `Option + Click` で複数の場所を同時に編集したり、`Command + /` でコメントアウトしたりできます。
3.  **Warp Driveでの共有**: チームや自分の他マシンとワークフローを同期します。

#### **③ 環境変数の管理**
Warpは標準のシェル（zsh/bash）の設定を引き継ぎますが、独自の管理機能が強力です。

*   **基本設定**: `.zshrc` や `.bashrc` に記述した `export` はそのまま有効です。
*   **Warp Drive (Environment Variables)**: `Command + 5` からプロジェクトごとに変数をセットできます。
*   **機密情報の保護 (Dynamic Variables)**: APIキーなどを 1Password などの外部ツールから動的に取得するように設定でき、ローカルに `.env` を置く必要がなくなります。
*   **direnv の併用**: フォルダ移動時に自動で環境変数を切り替えたい場合は、`direnv` をインストールしてWarp内でフックするのが一般的です。
