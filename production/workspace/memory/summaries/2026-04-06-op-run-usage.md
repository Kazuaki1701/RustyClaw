K様、`op run` コマンドについてですね。承知いたしました！。

`op run` は、**1Password に保存されている機密情報を、環境変数として一時的に展開しながらコマンドを実行する**ための非常に強力なツールです。これを使えば、ローカル環境に `.env` ファイルなどの生データを保存する必要がなくなります。

### 基本的な仕組み
コマンドの実行中だけメモリ上に環境変数を展開し、コマンドが終了するとその情報は消滅します。

```bash
# 基本的な書式
op run -- [実行したいコマンド]
```

### 具体的な使い方

1.  **環境変数の参照を記述したファイル（.envなど）を用意する**
    ファイル内には実際のトークンではなく、1Passwordの「シークレット参照（Secret Reference）」を記述します。
    ```text:example.env
    STRIPE_API_KEY="op://Private/Stripe/api-key"
    DB_PASSWORD="op://Production/Database/password"
    ```

2.  **`op run` でコマンドを実行する**
    `--env-file` オプションでファイルを指定して実行します。
    ```bash
    op run --env-file=example.env -- npm start
    ```
    これで、プログラム内からは `process.env.STRIPE_API_KEY` を通じて、1Password内の最新の値を安全に参照できます。

### 主なメリット
-   **セキュリティ**: 物理的なファイルに秘密鍵が残らないため、誤って Git にコミットするリスクをゼロにできます。
-   **一元管理**: 1Password 上で値を更新すれば、チーム全員（またはご自身の複数デバイス）で即座に最新の値が反映されます。
-   **透過性**: 既存のアプリケーションコードを一切書き換えることなく導入可能です。
