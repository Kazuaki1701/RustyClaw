# Discord疎通確認とgog環境調査

## TL;DR
Discordの接続テストを通じてBotの疎通を確認し、オンラインステータスが表示されない原因やGoogle Workspaceツール（gog）の未設定状況を調査しました。サンドボックス環境でのツール不足を特定し、解決に向けた設定変更案を提示しました。

## Topics
- **Discord接続確認**: ユーザーからのテストメッセージが正常に受信できていることを複数回にわたり確認しました。
- **ステータス表示の解説**: Botがオンライン表示にならない理由として、Discordのインテント設定やシステムの接続仕様を説明しました。
- **gogツールの欠落特定**: カレンダー取得の試行により、サンドボックス環境内にgogコマンドが存在しないことが判明しました。
- **環境構築の改善提案**: gogコマンドを動作させるためのホストディレクトリのマウントやDockerfileの修正案を提示しました。

Discord上でBotのステータスが「オンライン（緑色の丸🟢）」にならない場合、いくつか理由が考えられます。よくある原因をいくつかリストアップしてみますね。

**1. Discord Developer Portal の設定（Presence Intent）**
Botを作成した [Discord Developer Portal](https://discord.com/developers/applications) の設定で、Presence（プレゼンス）情報の取得・送信が許可されていない可能性があります。
* 左メニューの `Bot` > `Privileged Gateway Intents` の中にある **`Presence Intent`** が **ON** になっているか確認してみてください。

**2. GeminiClaw の接続仕様・ステータス設定**
GeminiClaw（私を動かしているシステム）のDiscord連携は、常に通信を開きっぱなしにするのではなく、必要なメッセージの送受信の時だけ動く仕組みになっている場合や、システム側で明示的にステータスを「オンライン」として送信していない場合があります。そのため、メッセージには即座に反応できても、見た目上はオフラインのままになることがあります。

**3. バックグラウンドプロセスの状態**
現在私はしっかり動いており、メッセージも受信できていますが、システム（PicoClawが管理しているホスト側のデーモン）がアイドリング（待機）状態に入っていると、Discord側から見て「オフライン」と判定されることがあります。
