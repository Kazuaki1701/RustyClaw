# YOLP気象情報APIリファレンスに基づく雨雲パトロール（天気予報）設計仕様

本ドキュメントは、Yahoo! Open Local Platform (YOLP) の「気象情報API」仕様をリファレンスとし、自律エージェントの Heartbeat Step 4（天気チェック）における**「ピンポイント雨雲パトロール・傘持ち出し声掛けアルゴリズム」**の設計と実装方針を定義する仕様書である。

---

## 1. 目的とユースケース

ユーザーの現在地（経度・緯度）を起点とし、**「今から1時間以内に雨が降るか」「どのくらいの強さで、いつ降り始めるか」**を HeartbeatService が定時（または数時間ごと）に自発チェックし、必要に応じて Discord に親しみやすい口調で「傘の持ち出し指示」や「洗濯物の取り込み警告」を通知する。

### ユースケース
*   **定時パトロール**: 朝 07:30 の Heartbeat 実行時、これから通勤ルートで雨が降る予測があれば「傘を持っていくと安心です」と声掛け。
*   **雨雲接近通知 (自発発言)**: 現在地付近で「あと20分後に本格的な雨 (降水強度 1.0mm/h 以上) が降り始める」予測を検知した場合、緊急アラートとして Discord に自発投稿する。

---

## 2. リファレンスAPI仕様 (YOLP 気象情報API 構造)

YOLP 気象情報API の以下のパラメータおよびデータ設計をモデルとする。

### リクエストパラメータ
*   `coordinates`: `経度,緯度` (例: `139.767,35.681` — 大森駅付近、本厚木駅付近などを `USER.md` から抽出)
*   `output`: `json`

### レスポンスデータ構造 (リファレンス)
指定座標における「現在（実測）」および「10分ごと、60分先まで」の降水強度（mm/h）の配列。
```json
{
  "Feature": [
    {
      "Name": "地点(139.69,35.69)の天気情報",
      "Property": {
        "WeatherList": {
          "Weather": [
            { "Type": "observation", "Date": "202605302120", "Rainfall": 0.0 }, // 実測
            { "Type": "forecast",    "Date": "202605302130", "Rainfall": 0.2 }, // 10分後予測
            { "Type": "forecast",    "Date": "202605302140", "Rainfall": 1.5 }, // 20分後予測
            { "Type": "forecast",    "Date": "202605302150", "Rainfall": 2.0 }, // 30分後予測
            { "Type": "forecast",    "Date": "202605302200", "Rainfall": 0.5 },
            { "Type": "forecast",    "Date": "202605302210", "Rainfall": 0.0 },
            { "Type": "forecast",    "Date": "202605302220", "Rainfall": 0.0 }
          ]
        }
      }
    }
  ]
}
```

## 3. 実装用代替API・データソースの選定

YOLP 気象情報API（終了）の完全な互換データおよび予測アルゴリズムを実現するため、以下のいずれかを採用する。

### 候補 A: Open-Meteo API（無料・APIキー不要・推奨）
緯度経度からピンポイントの短期予測を JSON で取得可能。日本国内もカバーしており、15分ごとの高解像度の降水予測が取得可能。
*   **エンドポイント**: `https://api.open-meteo.com/v1/forecast`
*   **クエリパラメータ**: `latitude=35.69&longitude=139.69&minutely_15=precipitation&timezone=Asia%2FTokyo`
*   **メリット**: クレジット表記のみで利用可能、極めて安定、JSON パースが容易。

### 候補 B: 気象庁 高解像度降水ナウキャスト（JSONデータ）
日本国内の最高精度の気象レーダー情報。
*   **エンドポイント**: 気象庁が提供するナウキャストタイルの JSON / 画像データ。
*   **デメリット**: データ形式が複雑であり、インプロセスでの座標計算とデコードのオーバーヘッドが大きい。

## 4. エージェント雨雲警報アルゴリズム (Rain Patrol)

`rustyclaw-tools` に `YolpWeatherTool` (または `OpenMeteoWeatherTool`) を実装し、以下のアルゴリズムで判定を行う。

### ① 降水強度の判定閾値としきい値
エージェントは取得した 60分先までの `Rainfall` (降水強度 mm/h) を走査し、最大の強度に基づいてアラートの種類を決定する。

| 降水強度 (mm/h) | 天気状況 | エージェントの対応・声掛けトーン |
|---|---|---|
| **`0.0`** | 降水なし（晴れ/曇り） | 無音 (HEARTBEAT_OK)。余計な通知はしない。 |
| **`0.1 〜 1.0`** | 弱い雨（ポツポツ） | 折りたたみ傘の推奨。「これから弱い雨が降るかもしれません」 |
| **`1.0 〜 5.0`** | 本格的な雨（ザーザー） | 傘の携帯・洗濯物取り込み推奨。「雨になります。傘をお持ちください」 |
| **`5.0 以上`** | 強い雨・豪雨 | 警戒アラート。「土砂降りの雨になります。外出はお気をつけください」 |

### ② 二重通知の防止ロジック（最重要）
Heartbeat は 10〜30 分の間隔で高頻度に起動するため、雨が降り続いている間、毎回「雨です」と通知し続けるとノイズになる。

*   **解決策**: SQLite `seen_items` テーブル（または `patrol_state`）に、最後に雨アラートを送信したタイムスタンプ (`last_rain_alert_at`) を記録する。
*   **ロジック**:
    1.  前回のアラート送信から **180分（3時間）以内** の場合は、雨を検知しても Discord への新規通知はスキップする（silent）。
    2.  ただし、降水強度が「小雨（0.5mm/h）」から「豪雨（6.0mm/h）」へ急激に悪化するなどの「警報レベルの上昇」を検知した場合は、インターバル期間内であっても例外的に割り込み通知を行う。

### ③ Quiet Hours（深夜帯）の動作
*   深夜 **23:00 〜 08:00** の間は、激しい豪雨（避難レベル）を除き、Discord へのメッセージ送信は行わない。
*   ただし、パトロールの動作記録として `patrol/findings.md` には `[HH:MM] Rain predicted (skipped due to quiet hours)` として記録する。
