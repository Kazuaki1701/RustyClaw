# 気象庁・Open-Meteoハイブリッド天気予報に基づく雨雲パトロール（天気予報）設計仕様

本ドキュメントは、**「気象庁API」および「Open-Meteo API」を組み合わせたハイブリッド天気予報システム**、および自律エージェントの Heartbeat Step 4 における**「ピンポイント雨雲パトロール・傘持ち出し声掛けアルゴリズム」**の設計と実装方針を定義する仕様書である。

---

## 1. 目的とユースケース

ユーザーの現在地（経度・緯度）を起点とし、**「今から1時間以内に雨が降るか」「どのくらいの強さで、いつ降り始めるか」**を HeartbeatService が定時（または数時間ごと）に自発チェックし、必要に応じて Discord に親しみやすい口調で「傘の持ち出し指示」や「洗濯物の取り込み警告」を通知する。

### ユースケース
*   **定時パトロール**: 朝 07:30 の Heartbeat 実行時、これから通勤ルートで雨が降る予測があれば「傘を持っていくと安心です」と声掛け。
*   **雨雲接近通知 (自発発言)**: 現在地付近で「あと20分後に本格的な雨 (降水強度 1.0mm/h 以上) が降り始める」予測を検知した場合、緊急アラートとして Discord に自発投稿する。

---

## 2. 気象庁・Open-Meteo ハイブリッドAPI仕様

本天気予報システムでは、以下の2つのAPIから得られるデータをマージして利用する。

### 1. 気象庁予報 API (`weather.tsukumijima.net`)
*   **用途**: 日本語の広域天気予報（テロップ、最高・最低気温、天気概況テキスト、降水確率）の取得。
*   **リクエスト**: 地域コード（例: 大森の場合は `130010` 等）を指定してフェッチ。

### 2. Open-Meteo API (`api.open-meteo.com`)
*   **用途**: 指定されたピンポイント座標（緯度・経度）における「直近60分間の15分刻み降水量予測（mm）」の取得。
*   **リクエスト**: `latitude={lat}&longitude={lon}&minutely_15=precipitation&timezone=Asia%2FTokyo` を指定してフェッチ。

### データマージ処理
スクリプト `504_get-weather.sh` 内部で、引数 `city` コードから緯度（`lat`）と経度（`lon`）へのマッピングを行い、両APIを多重フェッチする。`jq` を用いて、気象庁の広域予報データに Open-Meteo の `rain_next_60min` 配列を合流させた JSON を出力する。

#### 出力 JSON 形式
```json
{
  "location": "OMORI",
  "telop": "曇時々晴",
  "today_max_c": 25,
  "today_min_c": 18,
  "weather_detail": "くもり　昼過ぎから夕方晴れ",
  "wind": "南の風",
  "chance_of_rain": { "T00_06": "--%", "T06_12": "10%", ... },
  "rain_next_60min": [
    { "time": "06:45", "mm": 0.0 },
    { "time": "07:00", "mm": 0.4 },
    { "time": "07:15", "mm": 1.2 },
    { "time": "07:30", "mm": 0.0 }
  ],
  "forecast_text": "..." 
}
```

## 3. エージェント雨雲警報アルゴリズム (Rain Patrol)

エージェントは取得した 60分先までの `rain_next_60min` (降水量 mm/15min) を走査し、最大の雨量に基づいてアラートの種類を決定する。

### ① 降水強度の判定閾値としきい値

| 降水量 (mm/15min) | 天気状況 | エージェントの対応・声掛けトーン |
|---|---|---|
| **`0.0`** | 降水なし（晴れ/曇り） | 無音 (HEARTBEAT_OK)。余計な通知はしない。 |
| **`0.1 〜 0.5`** | 弱い雨（ポツポツ） | 折りたたみ傘の推奨。「これから弱い雨が降るかもしれません」 |
| **`0.5 〜 1.5`** | 本格的な雨（ザーザー） | 傘の携帯・洗濯物取り込み推奨。「雨になります。傘をお持ちください」 |
| **`1.5 以上`** | 強い雨・豪雨 | 警戒アラート。「土砂降りの雨になります。外出はお気をつけください」 |

### ② 二重通知 of 防止ロジック（最重要）
Heartbeat は 10〜30 分の間隔で高頻度に起動するため、雨が降り続いている間、毎回「雨です」と通知し続けるとノイズになる。

*   **解決策**: SQLite `seen_items` テーブル（または `patrol_state`）に、最後に雨アラートを送信したタイムスタンプ (`last_rain_alert_at`) を記録する。
*   **ロジック**:
    1.  前回のアラート送信から **180分（3時間）以内** の場合は、雨を検知しても Discord への新規通知はスキップする（silent）。
    2.  ただし、降水強度が「小雨」から「豪雨」へ急激に悪化するなどの「警報レベルの上昇」を検知した場合は、インターバル期間内であっても例外的に割り込み通知を行う。

### ③ Quiet Hours（深夜帯）の動作
*   深夜 **23:00 〜 08:00** の間は、激しい豪雨（避難レベル）を除き、Discord へのメッセージ送信は行わない。
*   ただし、パトロールの動作記録として `patrol/findings.md` には `[HH:MM] Rain predicted (skipped due to quiet hours)` として記録する。
