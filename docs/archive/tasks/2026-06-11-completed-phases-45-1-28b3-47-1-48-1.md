## v0.4 context-mode 統合と cron 基盤整備

### Phase 45-1: ctx_index / ctx_search エージェント統合
- **完了日**: 2026-06-11
- **概要**: Phase 01 で `ingest_session_summary` を削除したまま代替未実装だったエピソード記憶蓄積の穴を埋めた。`rustyclaw-gateway/src/lib.rs` に `try_ctx_search` / `try_ctx_index` の fail-open ヘルパーを追加。Heartbeat ではプロンプト生成前に `try_ctx_search` で過去文脈を取得して注入し、Session-summary 生成成功後に `try_ctx_index` でエピソード記憶に登録する。両操作とも失敗時は `warn!` ログのみで Pipeline を停止しない（fail-open）。`execute_heartbeat()` の不要な `_rag_query: Option<&str>` 引数を除去。
- **関連計画書**: なし（インラインで実装）

### Phase 28b-3: Dashboard LANE QUEUE 表示名フォーマット変更

- **関連計画書**: なし（インラインで実装）

### Phase 48-1: croner crate 置き換え
- **完了日**: 2026-06-11
- **概要**: `crates/rustyclaw-gateway/src/cron.rs` の `next_run_epoch()` "cron" ブランチを `croner 3.0.1` に移行。`"HH:MM"` 形式を `"MM HH * * *"` の 5 フィールド cron 式に変換して `Cron::from_str().find_next_occurrence(&now, false)` で次回時刻を取得。同ファイルのループ④でサービス停止後の catch-up バグも修正（`== `比較 → `>=` 比較）。`rustyclaw-gateway/Cargo.toml` に `croner = "3.0.1"` を追加。
