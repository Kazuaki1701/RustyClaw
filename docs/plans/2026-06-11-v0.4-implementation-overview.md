# RustyClaw v0.4 Implementation Plan (Phase 01–03 Overview)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** v0.4 仕様（docs/specs/v0.4/00_rustyclaw.md）を 3 フェーズで段階的に実装し、最終的に context-mode をフル活用する状態にする。

**Architecture:**  
Phase 01 で RAG/Embedding 関連コードを全削除してコンパイルを通す（最小 Chat 可能状態）。
Phase 02 で ExternalMcpController を実装し context-mode を子プロセスとして起動、ctx_search / ctx_index をエージェントが利用可能にする。
Phase 03 で ctx_execute / ctx_patch を追加し context-mode をフル活用する。

**Tech Stack:** Rust/Tokio, rmcp 1.7, rig-core 0.38, SQLite (rusqlite), context-mode (Node.js ≥ 22.5)

```rust
// crates/rustyclaw-gateway/src/external_mcp.rs
use std::path::{Path, PathBuf};
use tokio::process::{Child, ChildStdin, ChildStdout};
use anyhow::{Context, Result};

pub struct ExternalMcpController {
    child: Child,
    workspace_root: PathBuf,
}

impl ExternalMcpController {
    /// context-mode 子プロセスを起動し stdin/stdout を返す。
    /// 呼び出し元は返された (stdin, stdout) を McpClientHandler.connect() に渡す。
    pub fn spawn(workspace_root: &Path) -> Result<(Self, ChildStdin, ChildStdout)> {
        let storage_dir = workspace_root.join(".context-mode");
        let mut child = tokio::process::Command::new("context-mode")
            .env("CONTEXT_MODE_DIR", storage_dir.to_string_lossy().as_ref())
            .env("CONTEXT_MODE_PLATFORM", "custom-rustyclaw")
            .stdin(std::process::Stdio::piped())
            .stdout(std::process::Stdio::piped())
            .stderr(std::process::Stdio::inherit())
            .spawn()
            .context("ExternalMcpController: context-mode の起動に失敗")?;
        let stdin = child.stdin.take().context("stdin が取得できない")?;
        let stdout = child.stdout.take().context("stdout が取得できない")?;
        Ok((Self { child, workspace_root: workspace_root.to_path_buf() }, stdin, stdout))
    }

    /// 子プロセスの終了を待機する（プロセスが死んだことの検知に使用）。
    pub async fn wait(&mut self) -> std::io::Result<std::process::ExitStatus> {
        self.child.wait().await
    }

    pub fn workspace_root(&self) -> &Path {
        &self.workspace_root
    }
}
```

- [ ] **Step 2: connect_context_mode ヘルパー関数を追加**

既存 MCP 接続ブロック（line ~920-958）の直前に以下の関数を追加:

```rust
/// context-mode を起動し、クラッシュ時に指数バックオフで再接続するループを spawn する。
/// 接続確立後は tool_server_handle に ctx_execute / ctx_search / ctx_index / ctx_patch が自動登録される。
async fn start_context_mode(
    workspace_root: std::path::PathBuf,
    tool_server_handle: rig_core::tool::server::ToolServerHandle,
) -> tokio::task::JoinHandle<()> {
    tokio::spawn(async move {
        let mut delay = std::time::Duration::from_secs(1);
        loop {
            match external_mcp::ExternalMcpController::spawn(&workspace_root) {
                Err(e) => {
                    tracing::error!("context-mode 起動失敗: {e}");
                }
                Ok((mut ctrl, stdin, stdout)) => {
                    let handler = rig_core::tool::rmcp::McpClientHandler::new(
                        rmcp::model::ClientInfo::default(),
                        tool_server_handle.clone(),
                    );
                    match handler.connect((stdout, stdin)).await {
                        Ok(service) => {
                            tracing::info!("context-mode 接続完了 (ctx_execute/search/index/patch 登録)");
                            delay = std::time::Duration::from_secs(1);
                            tokio::select! {
                                _ = ctrl.wait() => {
                                    tracing::warn!("context-mode プロセスが終了。再起動します...");
                                }
                                result = service.waiting() => {
                                    tracing::warn!("context-mode MCP セッション終了 ({result:?})。再起動します...");
                                }
                            }
                        }
                        Err(e) => {
                            tracing::error!("context-mode MCP 接続失敗: {e}");
                        }
                    }
                }
            }
            tracing::info!("context-mode を {:?} 後に再起動...", delay);
            tokio::time::sleep(delay).await;
            delay = (delay * 2).min(std::time::Duration::from_secs(60));
        }
    })
}
```

```markdown
- `ctx_patch`: SEARCH/REPLACE 形式でファイルにパッチを適用する。大きなファイルの部分書き換えに適している。
```
